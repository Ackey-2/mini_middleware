# Phase 5 QoS Implementation Plan

> **For agentic workers:** TDD, one commit per task. Steps use checkbox (`- [ ]`).

**Goal:** Add DDS-style QoS (reliability + history depth), negotiate compatibility at discovery time, and fold reliability into data-plane routing.

**Architecture:** `Qos{reliability, history, depth}`. Discovery advertises `EndpointInfo.reliability`; the matcher enforces RxO (writer ≥ reader) and skips incompatible pairs. Subscriber queues become bounded (KEEP_LAST drops oldest). `DataPlane::use_shm = same_host && subscriber BEST_EFFORT` — RELIABLE pairs use the in-order TCP path even on the same host.

**Spec:** [docs/superpowers/specs/2026-06-18-phase5-qos-design.md](../specs/2026-06-18-phase5-qos-design.md)
